Objetivo: preparar uma aplicação Flask existente para execução em Docker.

Não alterar a lógica da aplicação.
Não modificar funcionalidades.

Criar apenas os artefatos necessários para containerização:

1. Dockerfile para execução da aplicação Flask.
2. docker-compose.yml para subir a aplicação.
3. Expor corretamente a porta utilizada pela aplicação.
4. Garantir persistência dos dados caso exista banco SQLite.
5. Explicar os arquivos criados.
6. Informar os comandos para:
   - build da imagem
   - execução do container
   - validação da aplicação

O foco desta etapa é apenas containerização e preparação do ambiente para futuras práticas DevOps e DevSecOps.
